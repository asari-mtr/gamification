# 第6章　メカニクスのカタログ

!!! abstract "この章のねらい"
    実装時に「引き出し」から選べるよう、ゲーミフィケーションのメカニクスを6系統に整理して棚卸しする。各メカニクスについて、**何を狙うか・どの心理欲求（第2章）に効くか・誰に効くか（第4章）・落とし穴**をセットで示す。最後に、状況から適切なメカニクスを選ぶディシジョンツリーを置く。

!!! tip "使い方"
    このカタログは「全部盛り」のためのものではない。第5章の 6D で定義した**目標・行動・プレイヤー**に合うものだけを選ぶ。メカニクスは多いほど良いのではなく、**少数を深く**機能させるほうが効く。

## 6.1 達成系メカニクス

「上達した・やり遂げた」という**有能感**を中心に駆動する系統。アチーバー型に強く効く。

| メカニクス | 狙い | 主な心理欲求 | 落とし穴 |
|---|---|---|---|
| **バッジ・実績** | 達成の可視化、次の目標提示 | 有能感 | 乱発で価値低下、取得条件の不透明 |
| **レベル / XP** | 成長の実感、長期目標 | 有能感 | 意味のない数字、後半の中だるみ |
| **進捗バー** | ゴールの可視化、完了欲求 | 有能感・目標勾配 | 終わりが見えないと逆効果 |
| **チャレンジ / クエスト** | 行動の方向づけ、達成感 | 有能感・自律性 | 難易度のミスマッチ |
| **スター/ランク評価** | 質的な達成度の提示 | 有能感 | 最高評価以外を失敗と感じさせる |

!!! warning "達成系の最大の注意点"
    多くの設計が達成系（左脳・外発寄り）に**偏る**。達成系だけだと、ソーシャライザーやフリースピリットを取りこぼし、報酬を止めると動機も止まる。他系統と組み合わせること。

## 6.2 競争系メカニクス

他者との**社会的比較**で駆動する系統。プレイヤー/キラー型に効くが、第3章で見たとおり**両義的**。

| メカニクス | 狙い | 主な心理欲求 | 落とし穴 |
|---|---|---|---|
| **リーダーボード** | 順位による競争動機 | 社会的影響・有能感 | 下位の離脱、不正、固定化 |
| **トーナメント** | 期間限定の競争イベント | 社会的影響・希少性 | 勝者総取りで多数が疎外 |
| **PvP / 対戦** | 直接対決 | 社会的影響 | スキル差のマッチング問題 |
| **ベンチマーク比較** | 「平均より上/下」の提示 | 社会的証明 | 平均以下の意欲低下 |

!!! tip "競争系を健全に使う設計"
    リーダーボードの「下位が萎える」問題への定番対策:

    - **セグメント化** — 全体一位ではなく、近い実力同士のリーグに分ける（Duolingoのリーグ方式）。
    - **相対表示** — 「あなたの周辺順位」だけ見せ、トップとの絶望的な差を見せない。
    - **期間リセット** — 毎週リセットして「今からでも巻き返せる」状態を保つ。
    - **個人ベスト** — 他者ではなく**過去の自分**と競わせる（有能感を損なわない）。
    - **任意参加** — 競争したくない人がオプトアウトできる（自律性の尊重）。

## 6.3 協力・社会系メカニクス

他者との**つながり・協力**で駆動する系統。関係性欲求を満たし、ソーシャライザー/フィランソロピストに効く。競争系の弱点を補完する。

| メカニクス | 狙い | 主な心理欲求 | 落とし穴 |
|---|---|---|---|
| **チーム / ギルド** | 帰属意識、相互の動機づけ | 関係性 | フリーライダー、過度な同調圧力 |
| **協力ミッション** | 共通目標での結束 | 関係性・有能感 | 貢献度の不均衡への不満 |
| **贈与 / ギフト** | 互恵、つながりの強化 | 関係性 | スパム化、義務感 |
| **メンター / 師弟** | 教えることの満足、知識継承 | 関係性・有能感 | メンター側の負担 |
| **社会的承認（いいね等）** | 他者からの肯定 | 関係性 | 承認欲求の過剰刺激 |
| **コミュニティ / フォーラム** | 交流と帰属の場 | 関係性 | 運営コスト、荒れ |

!!! example "競争と協力の組み合わせ"
    「チーム対チーム」は競争と協力を両立させる強力な型。チーム内では協力（関係性）、チーム間では競争（社会的影響）が働き、個人戦の「下位が孤立する」問題を緩和する。Pelotonのチーム機能や、企業の部署対抗ウォーキングイベントなどが好例。

## 6.4 進行・収集系メカニクス

**所有・蓄積・前進**の感覚で駆動する系統。Octalysisの「所有と保有」「発展と達成」に対応。

| メカニクス | 狙い | 主な心理欲求 | 落とし穴 |
|---|---|---|---|
| **コレクション** | 収集・コンプリート欲求 | 所有・有能感 | 終わると離脱、収集疲れ |
| **ストリーク（連続記録）** | 継続の習慣化 | 損失回避 | 途切れた時の喪失・罪悪感で離脱 |
| **アンロック** | 次への期待 | 好奇心・有能感 | 解放ペースの設計ミス |
| **デイリー報酬** | 再訪の習慣化 | 損失回避・現在バイアス | 義務感・作業化 |
| **アバター / 着せ替え** | 自己表現、愛着 | 自律性・所有 | 本質的活動から乖離しうる |
| **仮想通貨 / ポイント残高** | 蓄積と交換の動機 | 所有 | インフレ、射幸性 |

!!! warning "ストリークの両刃性"
    ストリーク（連続記録）は損失回避を使った強力な継続装置だが、**途切れた瞬間に離脱**を招きやすい。対策として、「ストリーク・フリーズ（1日休んでも記録が途切れない救済）」（Duolingo）や、月単位の達成に切り替えるなど、**1日の失敗で全てを失わせない**設計が定石。過度な強迫を与えると、健康・学習の文脈では逆効果かつ倫理問題（第14章）になる。

## 6.5 経済系メカニクス

**仮想経済（通貨・取引・希少性）**で駆動する系統。深いエンゲージメントを生む反面、射幸性・倫理リスクが最も高い。

| メカニクス | 狙い | 主な心理欲求 | 落とし穴 |
|---|---|---|---|
| **仮想通貨** | 行動と報酬の循環 | 所有 | インフレ管理、換金性の法規制 |
| **取引 / マーケット** | プレイヤー間経済、関与深化 | 所有・社会的影響 | 不正・転売・格差 |
| **希少アイテム** | 価値と欲求の創出 | 希少性・所有 | 過度な煽り |
| **ガチャ / ルートボックス** | 変動報酬による強い渇望 | 予測不能性 | **射幸性・依存・法規制（第14章）** |
| **オークション / 入札** | 競争的な獲得 | 希少性・社会的影響 | 過熱、支出誘発 |

!!! danger "経済系・変動報酬の倫理的境界"
    ガチャ／ルートボックスは Skinner の変動比率強化（第2章）を直接利用しており、**極めて強力で、極めて危険**。複数国でギャンブル規制の対象になり、未成年保護の観点から確率表示の義務化なども進む。学習・健康・業務など、本来目的が「ユーザーの利益」である領域では**原則として避ける**。詳細は第14章。

## 6.6 物語・没入系メカニクス

**意味・世界観・感情移入**で駆動する系統。Octalysisの「叙事的意義」に対応し、活動に一貫した意味を与える。右脳・ホワイトハットで持続性が高い。

| メカニクス | 狙い | 主な心理欲求 | 落とし穴 |
|---|---|---|---|
| **ナラティブ / ストーリー** | 活動に意味と文脈を与える | 叙事的意義 | 取ってつけた物語は白ける |
| **テーマ / 世界観** | 没入感、一貫した体験 | 叙事的意義 | 本質的活動との不整合 |
| **キャラクター / マスコット** | 感情移入、ガイド役 | 関係性・叙事的意義 | 幼稚に見えるリスク |
| **ロールプレイ / 役割** | 主体性、なりきり | 自律性・叙事的意義 | 文脈に合わないと浮く |
| **ミッションの大義** | 「自分は意味あることをしている」 | 叙事的意義 | 誇張・欺瞞は不信を招く |

!!! example "物語が活動を変える例"
    ランニングアプリ「Zombies, Run!」は、走る行為に「ゾンビから逃げ、物資を集めて拠点を守る」という物語を与えた。同じ「走る」でも、物語がある体験はない体験よりはるかに没入的になる。物語系は、単調な反復活動（運動・学習・記録）に**意味の文脈**を与えるのに特に有効。

## 6.7 メカニクス選定ディシジョンツリー

「どれを使うか」を、状況から逆算するための簡易ガイド。第5章の 6D で目標・行動・プレイヤーを定義した後に使う。

!!! example "選定の問い（上から順に）"
    **Q1. 解決したいのは「継続・習慣化」か？**

    - → ストリーク（救済付き）、デイリー目標、プログレッションループ、即時フィードバック。

    **Q2. 「上達・スキル習得」を支えたいか？**

    - → 進捗バー、レベル/XP、適応的難易度、意味あるバッジ、個人ベスト。

    **Q3. ユーザーは「つながり・仲間」を求めているか？**

    - → チーム、協力ミッション、コミュニティ、メンター。（競争に逃げない）

    **Q4. 「短期の盛り上がり・イベント」を作りたいか？**

    - → トーナメント、期間限定チャレンジ、希少報酬。（ブラックハットは戻る理由とセット）

    **Q5. 活動が「単調・意味が見えにくい」か？**

    - → ナラティブ、テーマ、大義の可視化。

    **Q6. ユーザータイプが多様 or 不明か？**

    - → **複数系統を並立**（達成＋協力＋探索）。単一系統に賭けない（第4章 多動機設計）。

    **⚠️ 使ってはいけないかの最終チェック**

    - 変動報酬・ガチャ・過度な希少性を、ユーザー利益が目的の領域で使おうとしていないか？（第14章）
    - 競争系だけで、下位ユーザーを切り捨てていないか？

!!! check "メカニクス選定のチェック"
    - [ ] 各メカニクスが 6D の**目標・行動**に紐づいているか？
    - [ ] 各メカニクスを**第2章の心理欲求**で説明できるか？
    - [ ] **達成系（左脳）に偏って**いないか？ 右脳・社会系を補ったか？
    - [ ] 競争系を使うなら**下位対策**（セグメント・個人ベスト等）があるか？
    - [ ] ストリーク等の損失回避系に**救済**があるか？
    - [ ] 経済系・変動報酬の**倫理・法規制**を確認したか？
    - [ ] 数を絞り、**少数を深く**機能させているか？

## この章のまとめ

- メカニクスは6系統:**達成・競争・協力/社会・進行/収集・経済・物語/没入**。
- 各メカニクスは**心理欲求**と**ユーザータイプ**に対応づけて選ぶ。落とし穴も必ずセットで把握する。
- 達成系に偏りがち。**協力・物語など右脳系で補完**し、持続性を高める。
- 競争系は両義的——**セグメント化・個人ベスト・任意参加**で健全化する。
- ストリーク等の損失回避系は**救済**を、経済系・変動報酬は**倫理・法規制**を要確認。
- 多いほど良いわけではない。**少数を深く**、6Dの定義に合うものだけを選ぶ。

次章では、選んだメカニクスを「**時間軸の体験**」として設計する——オンボーディングから、難易度曲線、リテンション、エンドゲームまで。

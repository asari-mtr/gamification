# 第7章　体験の設計

!!! abstract "この章のねらい"
    メカニクスを「点」で置くのではなく、**時間軸に沿った一連の体験**として設計する。ユーザーが初めて触れる瞬間から、習熟し、定着し、やがて飽きるまで——各フェーズで何を設計すべきかを扱う。オンボーディング、難易度曲線、リテンション、報酬スケジュール、エンドゲーム、そして体験フロー図の描き方まで。

## 7.0 プレイヤージャーニーという視点

ユーザーの体験は時間とともに変化する。Amy Jo Kim の「プレイヤージャーニー」は、これを4フェーズで捉える[^kim2018]。

1. **発見（Discovery）** — なぜ始めるのか。最初の価値を感じる。
2. **オンボーディング（Onboarding）** — 使い方と楽しさを学ぶ。最初の成功体験。
3. **習熟（Scaffolding / Habit Building）** — 繰り返し、上達し、習慣になる。
4. **熟達（Mastery / Endgame）** — 達人として、あるいは指導者として関わり続ける。

各フェーズで、効くメカニクス（第6章）も、満たすべき欲求も異なる。「全ユーザーに同じ体験」ではなく、「**今このユーザーがどのフェーズにいるか**」で出し分けるのが体験設計の核心である。

## 7.1 オンボーディング設計 — 最初の数分を制する

ユーザーの大半は**最初の数分〜初日**で離脱する。オンボーディングは、ゲーミフィケーション施策の中で最も投資対効果が高い領域の一つ。

### 原則

- **早い成功（early win）** — 最初のレベルアップ・最初のバッジ・最初の「できた！」を**できるだけ早く**訪れさせる。有能感の即時供給（第2章）。
- **段階的開示（progressive disclosure）** — 全機能を一度に見せない。必要なものだけを順に解放し、認知負荷を下げる。
- **やりながら学ぶ** — 説明を読ませるより、実際に行動させてフィードバックで教える（Fogg の「行動を簡単に」）。
- **初期進捗の付与** — スタンプカードに最初の数個を埋めておく（目標勾配効果、第2章）。「もう始まっている」感覚。
- **価値の即時提示** — 「これを続けると何が得られるか」を最初の体験の中で実感させる。

!!! example "Duolingoのオンボーディング"
    Duolingoは、アカウント作成前にいきなり最初のレッスンを体験させ、即座に正誤フィードバックと小さな達成を与える。「学習目標の設定」で自律性を、「初日からのストリーク開始」で継続のフックを作る。説明より体験を先行させ、最初の成功を数十秒で届ける設計。

!!! danger "オンボーディングのアンチパターン"
    - 長いチュートリアルを最初に全部見せる（離脱を招く）。
    - 価値を感じる前に登録・課金を要求する。
    - 最初の成功までが遠い（最初のバッジまで何時間もかかる）。

## 7.2 難易度曲線とスキャフォルディング

フロー理論（第2章）が教えるとおり、**挑戦とスキルの均衡**を保ち続けることが没入の条件。ユーザーのスキルは上がるので、難易度も**動的に**上げ続ける必要がある。

### 難易度曲線の設計

- **緩やかな立ち上がり** — 序盤は易しく、成功体験を連続させる。
- **段階的な上昇** — スキル向上に合わせて少しずつ難しく。退屈（簡単すぎ）も不安（難しすぎ）も避ける。
- **緩急（リズム）** — 難所の後に易しい区間を置き、緊張と緩和のリズムを作る。一本調子は疲れる。
- **スキャフォルディング（足場かけ）** — 最初は補助（ヒント・ガイド）を厚くし、習熟に応じて外していく。教育学由来の概念。

### 適応的難易度（adaptive difficulty）

ユーザーの実力に応じて難易度を自動調整する仕組み。

- 正答率・達成スピード・離脱の兆候などから、難易度をリアルタイムに調整。
- うまくいけば、全ユーザーを**それぞれのフロー状態**に保てる。
- 第16章のAI/パーソナライゼーションが、この適応をより精緻にする。

!!! tip "難易度の自己選択"
    自動調整が難しければ、**難易度をユーザー自身に選ばせる**のも有効（自律性の尊重）。「やさしい/普通/挑戦的」を選べるだけで、多様なスキルレベルに対応でき、かつ自律性欲求を満たせる。

## 7.3 リテンション設計

定着（リテンション）は、習慣化（第2章）を時間軸で設計すること。

### 再訪のトリガー設計

- **デイリー目標 / デイリー報酬** — 「今日もやる理由」を作る（現在バイアス対策）。
- **ストリーク** — 継続のフック。ただし**救済（フリーズ）付き**で離脱を防ぐ（第6章）。
- **アポイントメント・メカニクス** — 「特定の時間に来ると良いことがある」（例:毎日決まった時刻のボーナス）。
- **通知（トリガー）** — Fogg の P（プロンプト）。ただし過剰な通知は自律性を損ない、通知オフ・離脱を招く。**価値ある通知だけ**を、適切な頻度で。

### 中長期のエンゲージメント

- **長期目標** — レベル、ティア、コレクションのコンプリートなど、数週間〜数か月先のゴール。
- **新規性の供給** — 新コンテンツ・新イベント・季節イベントで、ノベルティ効果の減衰（後述）に対抗する。
- **再エンゲージメント** — 離脱したユーザーを呼び戻す設計（「おかえりボーナス」、進捗の保全）。

!!! warning "ノベルティ効果の減衰"
    新しい仕掛けへの反応は時間とともに薄れる（**ノベルティ効果**）。導入直後に上がった指標が数週間で元に戻るのは典型的な現象。対策は、(1) 内発的動機に根ざした設計（右脳・ホワイトハット、第5章）にすること、(2) コンテンツ・イベントを継続的に更新すること。「一度作って終わり」では必ず減衰する（第13章で測定法を扱う）。

## 7.4 報酬スケジュールの設計

「いつ・どのように報酬を与えるか」は、行動の定着を大きく左右する。Skinner の強化スケジュール（第2章）が基礎理論。

| スケジュール | 内容 | 効果 | 用途 |
|---|---|---|---|
| **連続強化** | 毎回必ず報酬 | 習得は速いが、止めると消去も速い | 学習の初期、行動の定着前 |
| **固定間隔** | 一定時間ごと（毎日ログインボーナス） | 報酬直前に行動が集中 | デイリー報酬 |
| **固定比率** | 一定回数ごと（10回でバッジ） | 報酬後に一時停止しがち | コレクション、マイルストーン |
| **変動間隔** | 不定期な時間で報酬 | 安定した持続行動 | サプライズ報酬 |
| **変動比率** | 不定回数ごと（ガチャ） | **最も強力で消去されにくい**＝中毒的 | ※射幸性・倫理に最大注意 |

!!! tip "報酬設計の実務原則"
    - **初期は連続強化**で行動を定着させ、**徐々に間欠強化**へ移行すると、効率よく習慣化できる。
    - **変動比率（ガチャ的）**は最も強力だが、依存・射幸性のリスクが最大。ユーザー利益が目的の領域では避ける（第6章・第14章）。
    - 報酬は**情報的フィードバック**（有能感を伝える）として設計し、制御的（やらされ感）にしない。アンダーマイニング効果を回避する（第2章）。
    - **サプライズ報酬**（予期しない報酬）は、アンダーマイニング効果を起こしにくく、喜びも大きい。

## 7.5 エンドゲームと飽きへの対処

熟達したユーザー、やり尽くしたユーザーをどう繋ぎ止めるか。多くのサービスが見落とす「**天井問題**」。

### 天井に達したユーザーへの設計

- **無限の進行** — レベルキャップの先に「プレステージ」「シーズン」など、繰り返せる目標を用意。
- **熟達者の役割** — メンター、モデレーター、コンテンツ作成者など、**与える側**に回る道を用意（フィランソロピスト/ソーシャライザーへ、第4章）。SDTの関係性・有能感を新しい形で満たす。
- **横の広がり** — 別ジャンル・別スキルへの誘導（探索、フリースピリット向け）。
- **コミュニティ** — 活動そのものより、人とのつながりが定着の理由になる段階。

### 飽きの兆候と対策

- 兆候:利用頻度の低下、デイリーのスキップ、新コンテンツへの反応低下。
- 対策:新規性の供給、目標の再設定、季節イベント、ソーシャル要素の強化。
- **根本対策**は、外発的報酬だけに頼らず内発的動機（活動自体の面白さ・つながり・意味）を育てておくこと。

## 7.6 体験フロー図の描き方

設計を「点の集合」で終わらせないために、**時間軸×状態**で体験を可視化する。実務でそのまま使える描き方を示す。

!!! example "体験フロー図の構成要素"
    横軸に**プレイヤージャーニー**（発見→オンボーディング→習熟→熟達）を取り、各フェーズについて以下の行を埋める:

    | 行 | 問い |
    |---|---|
    | **ユーザーの状態・感情** | このフェーズで何を感じているか（不安・好奇心・達成感・飽き） |
    | **狙う心理欲求** | 有能感/自律性/関係性のどれを満たすか（第2章） |
    | **主要メカニクス** | このフェーズで効かせる仕掛け（第6章） |
    | **フィードバック** | 何を、どのタイミングで返すか |
    | **離脱リスクと対策** | このフェーズ特有の離脱要因と、その手当て |
    | **成功指標** | このフェーズの成否をどう測るか（第13章） |

    この表を埋めるだけで、「オンボーディングは厚いが習熟期がスカスカ」「全部達成系で関係性が空白」といった**設計の穴**が一目で見える。

!!! example "エンゲージメントループ図"
    各フェーズ内の短期循環を、**動機 → 行動 → フィードバック → （次の）動機** の円環として描く。各矢印について「何がユーザーを次に進ませるか」を書き込む。矢印が切れている箇所（フィードバックが弱い、次の動機がない）が、離脱ポイントになる。

!!! check "体験設計のチェック"
    - [ ] **最初の成功**が十分に早いか？（オンボーディング）
    - [ ] スキル上昇に合わせて**難易度が上がる**設計か？（フロー維持）
    - [ ] ストリーク等に**救済**があり、1回の失敗で離脱させないか？
    - [ ] 報酬は**情報的**（有能感を伝える）か、制御的（やらされ感）になっていないか？
    - [ ] **天井に達したユーザー**の行き先（エンドゲーム）があるか？
    - [ ] **ノベルティ減衰**への対策（内発的動機・継続更新）があるか？
    - [ ] 体験フロー図で、**全フェーズ**に欲求・メカニクス・対策が埋まっているか？（穴がないか）

## この章のまとめ

- 体験は**プレイヤージャーニー**（発見→オンボーディング→習熟→熟達）の時間軸で設計する。
- **オンボーディング**は最重要投資。早い成功・段階的開示・初期進捗で離脱を防ぐ。
- **難易度曲線**は、フロー維持のため動的に上げる。適応的難易度／自己選択を活用。
- **リテンション**はデイリー・ストリーク（救済付き）・長期目標・新規性供給で設計。
- **報酬スケジュール**は連続強化→間欠強化へ。変動比率は強力だが倫理に注意。情報的に与える。
- **エンドゲーム**を用意し天井問題に備える。熟達者には「与える側」の役割を。
- **体験フロー図**で全フェーズを可視化し、設計の穴を発見する。

次章では、ここまでの設計が陥りがちな**典型的な失敗パターン**を、回避のチェックリストとともに整理する。

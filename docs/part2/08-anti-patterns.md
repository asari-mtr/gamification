# 第8章　設計のアンチパターン

!!! abstract "この章のねらい"
    成功例から学ぶより、**失敗例から学ぶ**ほうが速いことが多い。本章では、ゲーミフィケーションが失敗する典型パターンを、原因と回避策とともに整理する。Gartner が「ゲーミフィケーション化アプリの80%は設計の稚拙さで失敗する」と警告した、その「稚拙さ」の正体を具体化する。最後にレビュー用チェックリストを置く。

## 8.1 ポインティフィケーション

最も多く、最も根深い失敗。**動機づけの設計を欠いたまま、PBL（ポイント・バッジ・ランキング）だけを貼り付ける**こと。

### なぜ起きるか

- 「ゲーミフィケーション＝ポイントを付けること」という誤解（第1章）。
- PBLは実装が簡単で、「やった感」が出るため、安易に選ばれる。
- 6Dのステップ6（要素の選定）から始めてしまう（第5章）。

### なぜ失敗するか

- ポイントやバッジ**自体**には意味がない。「何のためか」が伝わらなければ、ただの数字。
- 第2章のどの心理欲求も満たしていない。有能感・自律性・関係性が空白のまま。
- 短期的な物珍しさ（ノベルティ効果）が切れると、一気に無関心になる。

### 回避策

- **要素から始めない**。6Dで目標→行動→プレイヤー→ループ→楽しさを先に設計する（第5章）。
- 各PBLが「**どの心理欲求を、誰に対して満たすか**」を言えるようにする。
- ポイントより、**上達・達成・つながりの可視化**に重心を置く（第3章）。

## 8.2 外発的報酬による内発的動機の侵食

報酬を足したことで、かえって意欲が下がる失敗。**アンダーマイニング効果**（第2章）の現場版。

### 典型例

- もともと楽しんで書かれていた社内ナレッジ共有に「投稿ポイント」を導入したら、**ポイント目当ての薄い投稿が増え、質の高い自発的投稿が減った**。
- 趣味で続けていた活動に報酬を付けたら、報酬がないと続かなくなった。

### 回避策

- すでに内発的に楽しまれている活動には、**安易に外的報酬を足さない**。
- 報酬を使うなら**情報的フィードバック**（有能感を伝える）として設計し、制御的（やらされ感）にしない。
- **サプライズ報酬**を活用する（事前約束された報酬ほどアンダーマイニングを起こす）。
- 「報酬を卒業」して内発的動機で続く状態（内在化）を最終目標にする。

## 8.3 リーダーボードの逆効果

競争の万能視による失敗。リーダーボードが**大多数のユーザーの意欲を削ぐ**。

### なぜ失敗するか

- グローバルな単一ランキングでは、上位の一握り以外は「自分には届かない」と感じて離脱する。
- 上位が固定化すると、中位以下は競争を諦める。
- ソーシャライザー・フィランソロピスト（第4章）には競争自体が響かない、むしろ不快。
- 新規ユーザーが、既存の上位者に勝てず最初から心が折れる。

### 回避策（第6章の再掲＋）

- **セグメント化**（近い実力同士のリーグ）、**相対表示**（周辺順位のみ）、**期間リセット**。
- **個人ベスト**との競争（他者ではなく過去の自分）。
- **任意参加**（競争したくない人のオプトアウト）。
- そもそも競争が目標達成に必要かを問い直す。協力系で代替できないか（第6章）。

## 8.4 測定の歪み — 数字が目的化する

「測れるものを追わせる」ことで、本来の目的が歪む失敗。**グッドハートの法則**（「指標が目標になると、良い指標でなくなる」）の発現。

### 典型例

- コールセンターで「対応件数」を競わせたら、**一件あたりを雑に切り上げて件数を稼ぐ**ようになり、顧客満足が下がった。
- 学習アプリで「問題数」を報酬対象にしたら、**簡単な問題ばかりこなして難問を避ける**ようになった。
- 「ログイン日数」を追わせたら、ログインだけして何もしないユーザーが増えた。

### 回避策

- **数えやすい代理指標**ではなく、**本来の成果**に近い指標を設計する（第13章）。
- 量だけでなく**質**を評価軸に入れる。
- 単一指標に依存せず、複数指標でバランスを取る。
- 「この指標を最大化するために、ユーザーがしうる**最も怠惰な行動**は何か」を想像し、抜け穴を塞ぐ。

## 8.5 強制とやらされ感 — 自律性の剥奪

ゲーミフィケーションが「楽しい誘い」ではなく「**逃れられない義務**」になる失敗。SDTの自律性（第2章）を破壊する。

### 典型例

- 従業員に業務のゲーミフィケーションを**強制**し、ランキング下位を晒す→ 監視・圧力と受け取られ、士気低下。
- 過剰な通知・デイリー義務で、ユーザーが「やらされている」と感じる。
- ストリークが**強迫**になり、楽しみが苦痛に変わる。

### 回避策

- **任意性**を担保する（参加・通知・競争をオプトアウトできる）。
- 「やらないと罰」より「やると良いことがある」のフレーミング。
- とくに**労働文脈**では、監視・強制色が出ないよう細心の注意（第14章）。

## 8.6 その他の頻出アンチパターン

- **作りきり（運用の欠如）** — 一度作って放置。ノベルティ効果が切れて減衰（第7章）。継続的な更新・運用を前提に。
- **取ってつけた物語/テーマ** — 本質的活動と無関係な世界観は白ける。物語は活動に**意味の文脈**を与えるときだけ機能する（第6章）。
- **全部盛り** — メカニクスを詰め込みすぎて、何が楽しいのか分からなくなる。**少数を深く**（第6章）。
- **ターゲットの取りこぼし** — 単一ユーザータイプ（多くは競争・達成型）を前提にし、他を無視（第4章 多動機設計）。
- **本質的問題のすり替え** — UX・情報・能力の問題を、動機の問題と取り違えてゲーミフィケーションで覆い隠そうとする（第1章）。使いにくいプロダクトはバッジでは救えない。
- **ダークパターンへの接近** — 射幸性・損失回避・希少性の過剰利用が、操作・搾取に転化する（第14章で詳述）。

## 8.7 レビュー用アンチパターン・チェックリスト

設計レビュー・リリース前点検でそのまま使える。1つでも「はい」があれば赤信号。

!!! check "アンチパターン検出チェック（「はい」=危険信号）"
    **ポインティフィケーション**

    - [ ] 各PBLが「どの心理欲求を満たすか」を説明**できない**か？
    - [ ] 要素（バッジ等）から設計を始めた、あるいは目標が曖昧か？

    **内発的動機の侵食**

    - [ ] すでに楽しまれている活動に、外的報酬を足そうとしているか？
    - [ ] 報酬が「やらされ感（制御的）」を生んでいないか？

    **競争の逆効果**

    - [ ] グローバル単一ランキングで、下位対策が**ない**か？
    - [ ] 競争が苦手な層がオプトアウト**できない**か？

    **測定の歪み**

    - [ ] 追わせる指標を、ユーザーが**ズル**できる抜け穴があるか？
    - [ ] 量だけを評価し、**質**を見ていないか？

    **強制・自律性の剥奪**

    - [ ] 参加・通知が**強制**になっていないか？
    - [ ] 労働文脈で、監視・圧力と受け取られうるか？

    **運用・その他**

    - [ ] リリース後の**更新・運用計画がない**か？
    - [ ] 物語/テーマが本質的活動と**無関係**か？
    - [ ] メカニクスを**詰め込みすぎ**ていないか？
    - [ ] 本来は**UX・能力・情報**の問題を、動機の問題にすり替えていないか？

## この章のまとめ

- 失敗の典型は:**ポインティフィケーション／内発的動機の侵食／競争の逆効果／測定の歪み／強制とやらされ感**。
- 多くは「**要素から始める**」「**単一動機を前提にする**」「**運用しない**」という根の浅い設計に由来する。
- 競争・損失回避・希少性・変動報酬といった**強力な仕掛けほど、誤用すると有害**。第14章の倫理に直結する。
- リリース前に**アンチパターン・チェックリスト**で点検する習慣をつける。

これで第Ⅱ部（設計編）は完了。次の第Ⅲ部では、ここまでの設計原則が各分野——教育・ヘルスケア・ビジネス・社会——で**どう具体化され、何が成功し何が失敗したか**を、事例とともに見ていく。

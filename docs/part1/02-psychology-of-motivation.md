# 第2章　なぜ人は動くのか — 動機づけの心理学

!!! abstract "この章のねらい"
    ゲーミフィケーションの良し悪しは、結局のところ**人間の動機をどれだけ正しく理解しているか**で決まる。本章では、設計の土台となる主要な動機づけ理論——内発／外発、自己決定理論、フロー、行動経済学、習慣形成——を、実務の設計判断に翻訳できる形で整理する。最後に「理論→設計」の早見表を置く。

## 2.1 内発的動機づけと外発的動機づけ

動機づけは大きく二つに分けられる。

- **内発的動機づけ（intrinsic motivation）** — 活動それ自体が面白い・満足できるから行う。報酬は活動の「中」にある。例:パズルを解く快感、上達する喜び。
- **外発的動機づけ（extrinsic motivation）** — 活動の「外」にある結果（報酬・評価・罰の回避）のために行う。例:給料のため、罰金を避けるため。

素朴には「内発＝善、外発＝悪」と捉えられがちだが、これは正しくない。外発的動機づけにも**質の差**がある。自己決定理論（次節）は、外発的動機を「やらされ感の強いもの」から「自分の価値観に統合されたもの」まで連続体として捉える。

実務上の要点は次の2つ。

1. **持続性は内発的動機が強い。** 報酬を止めた途端にやめてしまう行動は、外発に依存しすぎている。長期的なエンゲージメントを狙うなら、内発的動機へ橋渡しする設計が要る。
2. **外発から始めて内発へ移行させる**設計はありうる。最初はポイントで誘導し、活動の面白さに気づいてもらい、最終的にポイントがなくても続く状態へ——という移行（内在化）が理想形。

## 2.2 自己決定理論（SDT）— 設計の中核理論

自己決定理論（Self-Determination Theory; Deci & Ryan）は、ゲーミフィケーション設計で**最も重要な理論的土台**である[^deci2000]。人間には3つの基本的心理欲求があり、これらが満たされるほど内発的動機が高まり、ウェルビーイングも向上するとされる。

### 3つの基本的心理欲求

| 欲求 | 意味 | 満たされると | 損なわれると |
|---|---|---|---|
| **有能感（Competence）** | 自分は上達している・うまくやれているという感覚 | 挑戦が楽しくなる | 無力感、回避 |
| **自律性（Autonomy）** | 自分の意志で選んでいるという感覚 | 主体的に関わる | やらされ感、反発 |
| **関係性（Relatedness）** | 他者とつながっている・受け入れられている感覚 | 帰属意識、継続 | 孤立、離脱 |

この3つは、ゲーミフィケーションの各メカニクスがなぜ効く/効かないかを説明する**共通言語**になる。

### SDTで読み解くメカニクス

- **有能感を支える** — 進捗バー、レベルアップ、達成バッジ、適切な難易度（フロー）、上達が見える可視化。
    - ただし「誰でも取れるバッジ」は有能感を生まない。**意味のある達成**であることが条件。
- **自律性を支える** — 選択肢の提供、自分のペース、目標の自己設定、アバターのカスタマイズ、「やってもやらなくてもよい」任意性。
    - 強制されたチャレンジや、断れない通知は自律性を**損なう**。
- **関係性を支える** — チーム、協力タスク、贈り合い、コミュニティ、共有。
    - 一方、ゼロサムの競争は関係性を損ないうる（勝者以外が疎外される）。

!!! warning "競争・ランキングの両義性"
    リーダーボードは有能感（上位なら）を満たす一方、下位のユーザーの有能感・関係性を**損なう**。SDTのレンズで見ると、競争は「効く人には効くが、多くの人を脱落させうる」両義的な仕掛けだとわかる。誰に対して何の欲求を満たしているのかを問うのが、SDT的設計思考である。

### 報酬がなぜ内発的動機を損ないうるか（CET）

SDTの下位理論である**認知的評価理論（CET）** は、外的報酬が内発的動機に与える影響を「情報的側面」と「制御的側面」で説明する。

- 報酬が**有能感の情報**として受け取られる（「よくやった」というフィードバック）→ 内発的動機を**高めうる**。
- 報酬が**行動を制御する手段**と受け取られる（「報酬のためにやらされている」）→ 自律性を奪い、内発的動機を**下げる**。

つまり、同じ「報酬」でも、**自律性を損なわず有能感を伝える形**にできるかが分かれ目になる。これがアンダーマイニング効果（2.6）の核心である。

## 2.3 フロー理論 — 挑戦とスキルの均衡

フロー（flow）は Csikszentmihalyi が提唱した、活動に完全に没入した状態である[^csik1990]。時間を忘れ、行為と意識が一体化する「ゾーン」とも呼ばれる体験。

### フローが生じる条件

フローの中核は、**挑戦の難しさ（challenge）とスキル（skill）の釣り合い**である。

- 挑戦 > スキル → **不安・ストレス**
- 挑戦 < スキル → **退屈**
- 挑戦 ≈ スキル（ともに高い水準で） → **フロー**

ユーザーのスキルは時間とともに上がるので、難易度も合わせて上げ続ける必要がある。一定難易度のままだと、最初は不安、やがて退屈に転落する。**難易度曲線の設計**（第7章）がここに直結する。

### フローの他の条件

- **明確な目標** — 今何を目指せばよいかが分かる。
- **即時のフィードバック** — 自分の行為がうまくいっているか即座に分かる。
- **集中を妨げない環境** — 不要な中断・ノイズがない。

!!! tip "設計への含意"
    ゲーミフィケーションでよく使う「適応的難易度（adaptive difficulty）」「段階的なアンロック」「即時フィードバック」は、すべてフロー状態を作り出すための仕掛けと理解できる。第16章のAIによるパーソナライゼーションは、この「挑戦とスキルの均衡」を個人ごとに自動調整する試みである。

## 2.4 行動経済学とナッジ

人間は完全に合理的には行動しない。行動経済学は、その**予測可能な非合理性**を扱う。ゲーミフィケーション設計で特に有用な概念を挙げる。

- **損失回避（loss aversion）** — 人は同額の利得より損失を強く嫌う（約2倍とされる）。「連続記録（ストリーク）を切らしたくない」心理はこれ。**獲得**より**失う恐れ**のほうが行動を強く駆動する。
- **現在バイアス（present bias）** — 将来の大きな利益より、目先の小さな報酬を過大評価する。健康・学習・貯蓄が続かない根本原因。即時の小さな報酬で「今やる理由」を作るのが対策。
- **目標勾配効果（goal-gradient effect）** — ゴールに近づくほど努力が増す。スタンプカードに最初から2個押してあると完了率が上がる（保有効果＋進捗の可視化）。進捗バーや「あと◯◯で達成」表示が効く理由。
- **アンカリング** — 最初に提示された数値が基準になる。目標値やレベル設計の見せ方に影響する。
- **社会的証明（social proof）** — 「他の人もやっている」が行動を促す。省エネ行動の促進などで実証（第12章）。
- **賦与効果（endowment effect）** — 自分のものになったと感じると手放したくなくなる。アバターやコレクションへの愛着、ポイント残高の維持動機。

### ナッジとの関係

ナッジ（nudge）は「選択を強制せず、より良い行動へそっと誘導する」設計思想[^thaler2008]。ゲーミフィケーションとナッジは重なる部分が大きいが、

- **ナッジ** — 主に選択アーキテクチャ（デフォルト設定、提示順、フレーミング）で**一回一回の選択**を誘導。
- **ゲーミフィケーション** — 進捗・達成・フィードバックの仕組みで**継続的なエンゲージメント**を作る。

両者を組み合わせると強力だが、どちらも「ユーザーの利益のための誘導」か「提供者の利益のための操作」かという**倫理の問題**を内包する（第14章）。

## 2.5 習慣形成のモデル

ゲーミフィケーションの多くは、最終的に**行動を習慣化する**ことを狙う。習慣化のメカニズムを理解しておく。

### Fogg Behavior Model（B=MAP）

BJ Fogg は、行動（Behavior）が起こるのは **動機（Motivation）・能力（Ability）・きっかけ（Prompt）** の3つが同時に揃ったときだとした[^fogg2009]。

$$ B = M \times A \times P $$

- 動機が低くても、**行動を簡単に（Aを上げる）** すれば実行されやすい。
- きっかけ（通知・トリガー）がなければ、動機と能力があっても行動は起きない。

設計上の最重要示唆は「**動機を上げるより、行動を簡単にするほうが効くことが多い**」という点。ゲーミフィケーションは動機（M）に効きがちだが、まず能力（A、＝行動のハードルを下げる）を見直すべき場面は多い。

### Hooked モデル（フック・モデル）

Nir Eyal の Hooked モデルは、習慣化プロダクトの4ステップ循環を示す[^eyal2014]。

1. **トリガー（Trigger）** — 外的（通知）→ やがて内的（退屈・不安などの感情）へ。
2. **アクション（Action）** — 報酬を期待して取る最小の行動。
3. **変動報酬（Variable Reward）** — 予測できない報酬。これが渇望を生む（Skinnerの変動比率強化に対応）。
4. **投資（Investment）** — ユーザーがデータ・労力・コンテンツを蓄積し、次のトリガーを呼び込み、離れにくくなる。

!!! danger "Hookedモデルの倫理"
    Hookedモデルは強力ゆえに危うい。Eyal自身が「依存（addiction）を作る技術」と「習慣（habit）を作る技術」の境界を問い、**ユーザーの利益になるか**という倫理テスト（"Manipulation Matrix"）を提示している。変動報酬と投資の組み合わせは、有益にも搾取的にもなりうる（第14章）。

## 2.6 アンダーマイニング効果 — 報酬が逆効果になるとき

**アンダーマイニング効果（undermining effect / overjustification effect）** とは、もともと内発的に楽しんでいた活動に外的報酬を与えると、かえって内発的動機が下がる現象である。

古典的な実験では、絵を描くのが好きな子どもに「上手に描いたら賞状をあげる」と報酬を約束すると、報酬のない子どもより、後に自由時間で絵を描かなくなった[^lepper1973]。「好きだから描く」が「報酬のために描く」に**置き換わって**しまったのである。

### いつ起きやすいか

- 報酬が**制御的**に感じられるとき（「やらされている」）。
- **期待された**（事前に約束された）有形の報酬。
- もともと**内発的動機が高い**活動。

### 設計上の対策

- すでに内発的に楽しまれている活動には、**外的報酬を安易に足さない**。
- 報酬を使うなら、**予期しない形**（サプライズ）や、**有能感を伝える情報的フィードバック**として設計する。
- ポイントそのものより、**上達・達成・つながりの可視化**に重心を置く。
- 最終的に報酬を「卒業」できる設計（内在化）を目指す。

!!! quote "実務の教訓"
    「とりあえずポイントを付ける」が危険なのは、コストの問題だけではない。**もともとあった内発的動機を破壊しうる**からだ。報酬は「使えば使うほど良い」ものではなく、慎重に扱うべき劇薬である。

## 2.7 理論を設計判断に翻訳する早見表

各理論は、現場では次のような問いとして使える。

| 理論 | 設計時に問うべきこと | 具体的な打ち手 |
|---|---|---|
| **自己決定理論（有能感）** | ユーザーは上達を実感できるか？ | 進捗可視化、適切な難易度、意味あるバッジ |
| **自己決定理論（自律性）** | ユーザーは自分で選べているか？ | 選択肢、自己目標設定、任意性、カスタマイズ |
| **自己決定理論（関係性）** | つながりを感じられるか？ | チーム、協力、共有、コミュニティ |
| **フロー理論** | 挑戦とスキルは釣り合っているか？ | 適応的難易度、段階的アンロック、即時FB |
| **損失回避** | 失う恐れを健全に使えているか？ | ストリーク（ただし過度な不安に注意） |
| **現在バイアス** | 「今やる理由」があるか？ | 即時の小報酬、デイリー目標 |
| **目標勾配効果** | ゴールが近く見えるか？ | 進捗バー、「あと◯個」表示、初期進捗 |
| **Fogg（B=MAP）** | 動機の前に、行動を簡単にできないか？ | 行動の細分化、摩擦の除去、適切なトリガー |
| **アンダーマイニング効果** | 報酬が内発的動機を壊していないか？ | 情報的FB重視、サプライズ報酬、報酬の卒業 |

!!! check "設計レビューの3つの基本問い（SDTベース）"
    どんなゲーミフィケーション施策も、まずこの3問で点検できる。

    1. これは**有能感**を高めるか、それとも無力感を生むか？
    2. これは**自律性**を尊重するか、それとも「やらされ感」を生むか？
    3. これは**関係性**を育むか、それとも孤立・疎外を生むか？

## この章のまとめ

- 動機には内発／外発があり、**持続性は内発が強い**。外発から内発へ橋渡しする設計が理想。
- **自己決定理論（有能感・自律性・関係性）** が設計の中核言語。あらゆる施策をこの3欲求で点検できる。
- **フロー**は挑戦とスキルの均衡から生まれ、難易度曲線設計の根拠になる。
- 行動経済学（損失回避・現在バイアス・目標勾配効果など）は、具体的な仕掛けの効きどころを教える。
- 習慣形成は **Fogg（B=MAP）** と **Hookedモデル**で捉える。「動機を上げるより行動を簡単に」が要点。
- **アンダーマイニング効果**——報酬は内発的動機を壊しうる劇薬。安易なポイント付与を戒める。

次章では、これらの動機づけを実現する「ゲームの構成要素」を分解し、PBLの限界とより深い設計フレームワークへ進む。

# 第4章　プレイヤーを知る — ユーザータイプ論

!!! abstract "この章のねらい"
    同じ仕掛けでも、人によって効き方は正反対になる。競争で燃える人もいれば、競争で萎える人もいる。本章では、ユーザーの動機の多様性を捉える代表的モデル（Bartle、HEXAD）を学び、「**誰にどのメカニクスが効くか**」を設計に組み込む方法を示す。最後にユーザータイプ別のメカニクス対応表を置く。

## 4.1 Bartleのプレイヤー4分類

Richard Bartle が多人数オンラインゲーム（MUD）の観察から導いた、最も古典的なプレイヤー類型である[^bartle1996]。2つの軸で4タイプに分ける。

- **軸1:行為の対象** — 他のプレイヤー（players）⇄ 世界・環境（world）
- **軸2:関わり方** — 働きかける（acting）⇄ 相互作用する（interacting）

|  | 世界に対して | 他者に対して |
|---|---|---|
| **働きかける（acting）** | **アチーバー（Achiever）** | **キラー（Killer）** |
| **相互作用する（interacting）** | **エクスプローラー（Explorer）** | **ソーシャライザー（Socializer）** |

- **アチーバー（達成者）** — ポイント・レベル・実績の収集、目標達成、コンプリートを好む。「勝つ」より「**達成する**」。
- **エクスプローラー（探検家）** — 仕組みの発見、隠し要素、知識の探求を好む。「**まだ知らないこと**」に動かされる。
- **ソーシャライザー（社交家）** — 他者との交流、協力、コミュニティを好む。**人とのつながり**が主目的。
- **キラー（競争者）** — 他者との競争、勝利、ランキング上位、相手を出し抜くことを好む。**他者に対する優位**を求める。

!!! warning "Bartleモデルの限界"
    Bartleモデルは MMORPG という特定文脈から生まれたもので、「キラー」概念などはゲーミフィケーション一般に直訳しにくい。また、人は単一タイプに固定されず、状況で揺れ動く。**入門の地図**としては優れているが、設計の精緻化には次節のHEXADのほうが向く。

## 4.2 HEXADモデル

Andrzej Marczewski が、ゲーミフィケーション向けに、しかも**自己決定理論（第2章）と接続する形で**整理した6タイプのモデル[^marczewski2015]。動機の源泉でユーザーを分ける点が実務的。

| タイプ | 主な動機（SDTとの対応） | 効くメカニクス |
|---|---|---|
| **フィランソロピスト（Philanthropist）** | 目的・意味（他者を助けたい） | 贈与、共有、メンター、意義ある貢献 |
| **ソーシャライザー（Socializer）** | 関係性（つながりたい） | チーム、ギルド、交流、協力 |
| **フリースピリット（Free Spirit）** | 自律性（自由に創造・探索したい） | 探索、カスタマイズ、自由な選択、クリエイティブツール |
| **アチーバー（Achiever）** | 有能感（上達・達成したい） | チャレンジ、レベル、バッジ、学習コンテンツ |
| **プレイヤー（Player）** | 外的報酬（見返りが欲しい） | ポイント、報酬、実績、抽選、交換 |
| **ディスラプター（Disruptor）** | 変化（既存を揺さぶりたい） | 投票、開発への影響、自由度、創発 |

### 内発タイプと外発タイプ

HEXADの肝は、上の6つを**内発的動機タイプ**と**外発的動機タイプ**に分けて捉える点にある。

- **内発的動機タイプ** — フィランソロピスト・ソーシャライザー・フリースピリット・アチーバー。それぞれ SDT の「目的（関係性の発展形）・関係性・自律性・有能感」に対応する。
- **外発的動機タイプ** — **プレイヤー**。報酬で動く。あらゆる人がもつ側面でもある。
- **横断タイプ** — **ディスラプター**。システムそのものに働きかけたい層。

!!! tip "なぜHEXADが実務向きか"
    HEXADは「タイプ」を SDT の心理欲求に対応づけているため、第2章の理論とそのまま接続できる。「アチーバーにはバッジ」と暗記するのではなく、「**有能感を求める人には達成系が効く**」と原理で理解できる。これにより、新しいメカニクスでも応用が利く。

## 4.3 動機の多様性とセグメンテーション

### 人は単一タイプではない

最重要の注意点。**現実のユーザーは複数タイプの混合**であり、しかも文脈で変化する。同じ人が、学習アプリではアチーバー、SNSではソーシャライザー、として振る舞うことは普通にある。タイプ分けは「人をラベルで固定する」ためではなく、「**動機の多様性を設計で取りこぼさない**」ためにある。

### 設計への含意 ― 単一動機に賭けない

ここから導かれる最も実践的な原則は、

> **特定のタイプにだけ効く仕掛けに全面依存しない。複数の動機経路を用意する。**

たとえばリーダーボード（競争＝プレイヤー/キラー型に効く）だけに頼ると、ソーシャライザーやフリースピリットを取りこぼし、しかも第3章で見たとおり下位ユーザーを離脱させる。これに対し、

- 競争（ランキング）＝ プレイヤー/アチーバー向け
- 協力（チーム目標）＝ ソーシャライザー向け
- 探索・カスタマイズ ＝ フリースピリット向け
- 貢献・共有 ＝ フィランソロピスト向け

を**並立**させれば、多様なユーザーがそれぞれの動機で関われる。これが「**多動機設計（multi-motivational design）**」の考え方である。

### ユーザータイプの把握方法

- **HEXAD ユーザータイプ調査** — Marczewski らによる質問紙でタイプ傾向を測定できる。研究や大規模サービスで利用。
- **行動データからの推定** — 実際の利用ログ（競争機能をよく使う/探索が多い等）からタイプ傾向を推定し、出し分ける（第16章の適応的ゲーミフィケーションへ）。
- **軽量な代替** — 厳密な測定が難しい場合でも、「複数の動機経路を用意する」という原則を守れば、多くは取りこぼさない。

## 4.4 文化・属性による反応の差

ユーザータイプに加え、より広い属性も効き方を左右する。

- **文化** — 競争や個人の達成の強調は、個人主義的文化では受け入れられやすいが、集団主義的文化では協力・集団の調和を重んじる傾向がある（Hofstede の文化次元論などが背景理論）。グローバル展開では同一設計が同じに効くとは限らない。
- **年齢** — 一般に若年層は競争・新規性への反応が強い傾向があるとされるが、個人差が大きく、年齢で決めつけるのは危険。
- **ジェンダー** — かつて「男性は競争、女性は協力」といった粗い一般化がされたが、実証的な裏付けは弱く、ステレオタイプに基づく設計は避けるべき。
- **ゲーム経験・リテラシー** — ゲームに親しんだ層は複雑なメカニクスを楽しめるが、不慣れな層には学習コストが障壁になる。オンボーディング（第7章）で吸収する。

!!! warning "ステレオタイプの罠"
    属性による傾向は**集団レベルの平均**であって、個人を決めつける根拠にはならない。属性ベースの決め打ちより、**行動データに基づく適応**（第16章）や、**複数経路を用意して本人に選ばせる**（自律性の尊重）ほうが、倫理的にも効果的にも優れる。

## 4.5 ユーザータイプ別メカニクス対応表

設計時の参照用。「このメカニクスは誰に効くか／誰を取りこぼすか」を一覧する。

| メカニクス | 強く効くタイプ | 効きにくい・逆効果になりうるタイプ |
|---|---|---|
| **ポイント・報酬・抽選** | プレイヤー、アチーバー | フリースピリット（自由を制約と感じる）、内発が強い層 |
| **バッジ・実績** | アチーバー、プレイヤー | フリースピリット（押し付けに感じうる） |
| **リーダーボード（競争）** | プレイヤー、（キラー的） | ソーシャライザー、フィランソロピスト、下位ユーザー全般 |
| **チーム・協力目標** | ソーシャライザー、フィランソロピスト | 単独志向の強い層 |
| **探索・隠し要素・収集** | フリースピリット、アチーバー | 即時報酬を求めるプレイヤー |
| **カスタマイズ・自己表現** | フリースピリット | 効率重視の層 |
| **贈与・メンター・他者貢献** | フィランソロピスト、ソーシャライザー | 競争志向の強い層 |
| **投票・開発への関与** | ディスラプター、フリースピリット | 受動的に楽しみたい層 |

## この章のまとめ

- **Bartleの4分類**（アチーバー・エクスプローラー・ソーシャライザー・キラー）は入門の地図。
- **HEXAD**（6タイプ）は SDT と接続し、実務に向く。内発タイプ4＋外発タイプ（プレイヤー）＋横断（ディスラプター）。
- 人は**単一タイプではなく混合**で、文脈で変化する。タイプ論は「動機の多様性を取りこぼさない」ために使う。
- 最重要原則は**多動機設計**——競争だけに賭けず、協力・探索・貢献など**複数の動機経路を並立**させる。
- 文化・属性の傾向は集団平均にすぎない。**決めつけより、選ばせる／適応する**。

これで第Ⅰ部（基礎編）は完了。次の第Ⅱ部では、ここまでの理論を**実装できる設計**へ落とし込む——フレームワーク、メカニクスのカタログ、体験設計、そしてアンチパターンへ進む。

[^marczewski2015]: Marczewski, A. (2015). *Even Ninja Monkeys Like to Play: Gamification, Game Thinking and Motivational Design*. CreateSpace.

# 第1章　ゲーミフィケーションとは何か

!!! abstract "この章のねらい"
    ゲーミフィケーションの定義を、流行語としてではなく**設計判断に使える概念**として捉え直す。隣接領域との違い、歴史的な系譜、現場でくり返される誤解を整理し、最後に「どんな課題に向き、どんな課題に向かないか」を判断するための観点を示す。

## 1.1 定義と本質

ゲーミフィケーション（gamification）の最も広く引用される定義は、Deterding らによる次のものである。

> **「ゲームデザイン要素を、ゲーム以外の文脈で用いること」**（the use of game design elements in non-game contexts）[^deterding2011]

短い定義だが、4つの語に設計上の含意が詰まっている。

- **ゲームデザイン要素（game design elements）** — ゲームの「丸ごと」ではなく、ポイント・バッジ・レベル・クエスト・フィードバックといった**部品**を借りる。完全なゲームを作るわけではない。
- **用いる（use）** — 表層の装飾ではなく、ユーザーの行動を促す**仕掛け**として機能させる。
- **ゲーム以外の（non-game）** — 娯楽そのものが目的のゲームと異なり、**別の本来目的**（学習・健康・業務など）が先にある。
- **文脈（contexts）** — 同じ要素でも、置かれる文脈によって効果は正反対になりうる。文脈依存性は本書を貫くテーマである。

### 本質は「動機づけのデザイン」

部品の話に引きずられると本質を見失う。ゲーミフィケーションの本質は、**人がある行動を「やりたい」「続けたい」と感じる状態を設計すること**にある。ポイントやバッジは目的ではなく、動機づけを生むための手段の一つにすぎない。

優れたゲームは、報酬で釣っているのではなく、**上達の実感・自分で選んでいる感覚・仲間とのつながり**といった、人間が本来もつ動機を巧みに引き出している。ゲーミフィケーションが目指すのも同じで、外側から飴を与えることではなく、**対象の活動そのものを魅力的にする**ことにある。この視点は第2章の動機づけ理論で深掘りする。

!!! quote "Kevin Werbach の視点"
    Werbach はゲーミフィケーションを「ゲームの要素とゲームデザインの技法を、ゲーム以外の文脈で使うこと」と定義し、**「要素」よりも「技法（デザインの考え方）」が重要**だと強調する[^werbach2012]。部品のリストではなく、プレイヤーの体験をどう設計するかという姿勢こそが核心だ、という立場である。

## 1.2 用語の整理 — 隣接概念との違い

ゲーミフィケーションは、ゲームを使った/ゲームに似た取り組みの一つにすぎない。混同されやすい隣接概念を区別しておくと、議論が明確になる。

| 概念 | 何か | ゲーム性 | 主目的 | 例 |
|---|---|---|---|---|
| **ゲーミフィケーション** | 既存の活動にゲーム**要素**を加える | 部分的 | 活動そのものの達成（学習・運動等） | 学習アプリの連続記録、歩数アプリのバッジ |
| **シリアスゲーム** | 完全な**ゲーム**を、娯楽以外の目的で作る | 完全 | 教育・訓練・啓発など | 手術トレーニング用ゲーム、防災シミュレーション |
| **ゲームベースド・ラーニング (GBL)** | 学習のために（既存/専用の）ゲームをプレイする | 完全 | 学習 | 歴史を学ぶための歴史ゲーム活用 |
| **遊び心のあるデザイン (playful design)** | 楽しさ・遊び心を加味するが明確なゲーム構造はない | なし〜断片 | 体験の質の向上 | 404ページの遊び、空港の足跡サイン |
| **プレイ (toys/playful)** | ルールや勝敗のない自由な遊び | ルールなし | 探索・表現 | レゴ、サンドボックス的UI |

Deterding らはこれを「**ゲーム/プレイ** の軸」と「**全体/部分** の軸」の2次元で整理した。横軸を「規則のある遊び（gaming）⇄ 自由な遊び（playing）」、縦軸を「全体（whole）⇄ 部分（parts）」とすると、ゲーミフィケーションは「**ゲーム寄り × 部分**」の象限に位置する。シリアスゲームは「ゲーム寄り × 全体」、遊び心のあるデザインは「プレイ寄り × 部分」に入る[^deterding2011]。

!!! tip "実務での使い分け"
    「フルスクラッチで完結したゲームを作る」のか、「既存サービスに要素を足す」のかで、必要な投資もリスクもまったく異なる。シリアスゲーム開発は数千万円規模になりうるが、ゲーミフィケーションは既存プロダクトへの**部分的な追加**として始められる。本書が主に扱うのは後者である。

## 1.3 歴史と系譜

「ゲーミフィケーション」という語は2000年代後半に広まった新しい言葉だが、**行動を動機づけるためにゲーム的な仕掛けを使う**という発想自体は古い。

### 前史 — ロイヤルティと行動科学

- **1896年・S&H グリーンスタンプ** — 買い物でスタンプを集め景品と交換する米国のロイヤルティプログラム。ポイント収集の原型。
- **1908年・ボーイスカウトのメリットバッジ** — 達成を可視化するバッジシステムの古典。
- **1981年・アメリカン航空 AAdvantage** — 近代的なマイレージプログラム。ステータス階層（ゴールド/プラチナ等）は今日のレベルデザインに直結する。
- **行動主義心理学** — Skinner の**強化スケジュール**（とくに変動比率強化）は、ゲームの報酬設計やガチャの心理に大きな影響を与えた（第7章）。

### 「ゲーミフィケーション」という語の登場

- **2002–2003年** — プログラマの Nick Pelling が "gamification" という語を作ったとされる（商業的には普及せず）。
- **2010年前後** — スマートフォンと位置情報サービスの普及が転機。**Foursquare**（2009）のチェックイン・バッジ・「メイヤー」称号が象徴的な成功例として注目され、語が一気に広まった。
- **2011年** — Deterding らの論文が学術的な定義を与え、Gartner が「2015年までに大企業の過半数がゲーミフィケーションを導入する」と予測。**ハイプ（過熱）期**に入る。
- **2012年** — Werbach & Hunter の "For the Win" や、Jane McGonigal の "Reality Is Broken" がビジネス・一般に概念を広めた。

### 幻滅から定着へ

過熱の後には反動が来た。2012年、ゲームデザイナーの Ian Bogost は安易なポイント付与を「**搾取ウェア（exploitationware）**」と痛烈に批判した[^bogost2011]。「ポイントとバッジを貼れば人が動く」という素朴な期待は、多くの失敗プロジェクトを生んだ。Gartner 自身も2012年、「**ゲーミフィケーション化されたアプリの80%は、設計の稚拙さゆえに目的を達成できないだろう**」と警告している[^gartner2012]。

しかしこの「幻滅期」を経て、分野はむしろ成熟した。流行語としての熱は冷めた一方で、

- 動機づけ心理学（とくに自己決定理論）に基づいた**理論的に正当な設計**、
- Duolingo・Strava・Apple Watch のアクティビティリングなど、**生活に定着した成功例**、
- 効果を検証する**実証研究の蓄積**（第15章）、

が積み上がった。今日のゲーミフィケーションは、「魔法のスパイス」ではなく、**動機づけを扱う設計分野の一つ**として位置づけられている。

## 1.4 よくある誤解と失敗の起点

現場での失敗の多くは、設計以前の**思い込み**に起因する。代表的な誤解を挙げる。

!!! danger "誤解1：ポイント・バッジ・ランキングを付ければゲーミフィケーション"
    最も根深い誤解。PBL（Points / Badges / Leaderboards）は**ゲーミフィケーションの構成要素のごく一部**にすぎない。動機づけの設計を欠いたままPBLだけ貼ることを、批判者は「**ポインティフィケーション（pointsification）**」と呼ぶ。詳しくは第3章・第8章。

!!! danger "誤解2：人は報酬で動く（外発的報酬への過信）"
    外発的な報酬は短期的には効くが、**もともと内発的に楽しんでいた活動への報酬は、むしろ意欲を下げる**ことがある（アンダーマイニング効果、第2章）。報酬の設計を誤ると、長期的な動機を破壊しうる。

!!! danger "誤解3：ゲーミフィケーションは万能で、どんな課題にも効く"
    効果は文脈・ユーザー・課題の性質に強く依存する。向かない場面で使えば逆効果になる（次節）。

!!! danger "誤解4：一度作れば終わり"
    新しさによる効果（**ノベルティ効果**）は時間とともに薄れる。リーダーボードやバッジへの反応は数週間で減衰することが多い。継続的な運用・更新を前提に設計する必要がある（第7章・第13章）。

!!! danger "誤解5：楽しければよい / 楽しさは飾り"
    逆方向の二つの誤解。「楽しさ」は目的ではなく、**本来目的（学習・健康・業務成果）を達成するための手段**である。一方で「真面目な目的だから楽しさは不要」というのも誤りで、楽しさの欠如は継続率を直接損なう。

## 1.5 実務者が最初に押さえるべき「向く課題・向かない課題」

ゲーミフィケーションに着手する前に、**そもそもこの課題に向いているか**を見極めることが、最も費用対効果の高い判断である。

### 向いている課題の特徴

- **行動の繰り返し・習慣化が価値を生む** — 学習、運動、記録、貯蓄など。継続そのものが成果に直結する領域。
- **進捗が可視化できる** — スキルの上達、達成度、累積など、フィードバックを返せる対象がある。
- **ユーザーに一定の自発性がある** — もともと「やったほうがいい」と思っているが続かない、という層に効きやすい。
- **行動と結果の間に時間差がある** — 効果が出るまで時間のかかる活動（語学・健康）では、途中のモチベーション維持に有効。
- **短期的なフィードバックを設計できる** — 即時の手応えを返せること。

### 向いていない/慎重を要する課題

- **本質的に内発的動機が強い活動** — すでに楽しんでいる活動に外発的報酬を足すと、意欲を損なうリスク（アンダーマイニング効果）。
- **一回限りの行動** — 繰り返しのない単発タスクでは、習慣化の仕組みが活きない。
- **創造性・質が重要で、量や速度を競わせると歪む活動** — ランキングや件数報酬が、手抜きや数字稼ぎを誘発する（第8章の「測定の歪み」）。
- **倫理的に強制色が出る場面** — 従業員の監視、断りにくい立場の人への適用など。エンゲージメントが**強制**に転化すると逆効果かつ不誠実（第14章）。
- **根本問題が動機ではなく能力・環境にある場合** — そもそも使いにくい、情報が足りない、といった問題はゲーミフィケーションでは解けない。UX・情報設計の問題を動機の問題とすり替えてはならない。

!!! check "着手前のセルフチェック"
    - [ ] 解決したい**行動**を、具体的に一文で言えるか？（例:「ユーザーが毎日5分学習する」）
    - [ ] その行動は**繰り返し**に価値があるか？
    - [ ] ユーザーは本当は「やりたい」と思っているか？（やりたくないことを無理にやらせようとしていないか）
    - [ ] 問題の原因は**動機**か、それとも能力・環境・UXか？
    - [ ] 短期的な**フィードバック**を返せる進捗指標があるか？
    - [ ] 数字を追わせることで、**質を損なう**副作用はないか？

このチェックで「動機の問題ではない」と分かれば、ゲーミフィケーション以前に解くべき課題がある、ということだ。逆に多くにチェックが付くなら、本書の以降の章が役に立つ。

## この章のまとめ

- ゲーミフィケーションとは「ゲームデザイン要素をゲーム以外の文脈で用いること」だが、本質は**動機づけのデザイン**にある。
- シリアスゲーム・GBL・遊び心のあるデザインとは区別される。ゲーミフィケーションは「ゲーム寄り×部分」の象限。
- 概念は新しいが系譜は古く、過熱と幻滅を経て、いまは**理論に基づく設計分野**として定着した。
- PBL万能論・報酬万能論・万能薬幻想・作りきり発想・楽しさ誤解という典型的な誤解が、失敗の起点になる。
- 着手前に「**この課題はそもそもゲーミフィケーションに向くか**」を見極めることが最重要。

次章では、「なぜ人は動くのか」を動機づけの心理学から掘り下げ、設計判断の土台を固める。

# 第11章　ビジネス・組織

!!! abstract "この章のねらい"
    ビジネス領域は、ゲーミフィケーションが最初に注目され、最も派手に失敗もした分野である。マーケティング（ロイヤルティ）、人事・従業員エンゲージメント、営業、クラウドソーシングを扱い、「動機づけ」と「労働における強制・搾取」の緊張関係——この分野固有の最大の論点——を正面から扱う。

## 11.1 マーケティング・ロイヤルティプログラム

ゲーミフィケーションの最も古く、最も確立された商業応用（第1章の歴史）。

- **ポイント・マイル・ティア** — 購買に応じたポイント、ステータス階層（ブロンズ〜ダイヤ）。第3章の進行システムそのもの。
- **スタンプカード** — 目標勾配効果（第2章）。「あと2個で1杯無料」。
- **会員ランク** — 上位ステータスの特別感（希少性・所有、第5章 Octalysis）。
- **アプリ内チャレンジ** — Starbucks の「特定商品を期間内に買うとボーナス星」など、行動を誘導するクエスト。

!!! example "Starbucks Rewards"
    星（ポイント）の獲得・ランク・期間限定チャレンジ・パーソナライズされたオファーを統合。購買頻度と単価を上げる商業的成功例。鍵は、(1) 報酬が**実利**（無料ドリンク）に直結し分かりやすい、(2) チャレンジが**パーソナライズ**され「自分向け」に感じられる（第16章）点。

!!! warning "ロイヤルティの落とし穴"
    - **割引依存** — 報酬目当ての顧客は、報酬がなくなると離れる（外発依存、第2章）。本当のロイヤルティ（ブランド愛着）とは別物。
    - **複雑化** — ティアやポイント規則が複雑すぎると、かえって離脱を招く。
    - **コスト** — ポイント原資の管理を誤ると採算が崩れる。

## 11.2 人事・従業員エンゲージメント

採用・研修・業務・エンゲージメント測定にゲーミフィケーションを使う領域。**最も慎重を要する**。

- **採用** — ゲーム形式のアセスメント、スキル評価。応募者体験の向上にも。
- **オンボーディング** — 新入社員の立ち上がりを、チェックリスト・進捗・バッジで支援。
- **研修・スキル開発** — 第9章の企業研修と重なる。
- **エンゲージメント施策** — 称賛（peer recognition）、表彰、社内チャレンジ。

!!! danger "労働文脈の根本的緊張"
    職場のゲーミフィケーションには、他分野にない固有の問題がある。**従業員は「やめる自由」が乏しい**——参加が事実上強制になりやすい。これはSDTの自律性（第2章）を根本から脅かす。

    - ランキングで下位を晒す → 公開羞恥・士気低下。
    - 業務量をポイント化して競わせる → 監視・締め付けと受け取られる。
    - 「楽しく見せた業務強化」が、**搾取の隠れ蓑**になりうる（第14章）。

    労働領域でのゲーミフィケーションは、**従業員の利益になるか、それとも管理・搾取の道具か**を厳しく問う必要がある。

### 健全に使うための原則

- **任意参加**を担保する。
- **個人の罰**（下位の晒し）ではなく**成長・称賛・協力**に重心。
- 業績評価・給与と直結させない（外発化・歪み、第8章）。
- 従業員を**設計に参加**させる（自律性、当事者性）。
- 監視ではなく**支援**のためであることを、運用で示す。

## 11.3 営業・コールセンター

成果が数値化しやすく、ゲーミフィケーションが多用される一方、**測定の歪み**（第8章）が最も顕在化しやすい領域。

- **典型施策** — 売上/件数ランキング、達成バッジ、チーム対抗、リアルタイムの目標達成可視化。
- **効果** — 短期的な数字の押し上げには効くことがある。

!!! warning "営業ゲーミフィケーションの歪み"
    - 「件数」を競わせると、**質を犠牲に件数を稼ぐ**（顧客満足の低下、強引な営業）。
    - 上位の固定化で、中位以下が競争を諦める（リーダーボードの逆効果、第8章）。
    - 数字のための行動が、**長期的な顧客関係を破壊**する。
    - グッドハートの法則:数字が目標になると、その数字は成果の良い指標でなくなる。

    対策:**質の指標を併用**、**個人ベスト・チーム協力**を取り入れる、**短期の数字ゲームに長期目標を接続**する。

## 11.4 クラウドソーシング・市民科学

不特定多数の自発的貢献を、ゲーミフィケーションで引き出す領域。**内発的動機との相性**が良い成功例が多い。

- **Foldit** — タンパク質の折りたたみをパズルゲーム化。市民の集合知が、専門家でも解けなかった構造を解明した有名例。
- **Stack Overflow** — 評判ポイント・バッジ・特権で、専門家の自発的な回答を促進。質の高い知識共有を維持。
- **Wikipedia / OpenStreetMap** — バッジや編集回数の可視化が、貢献の一部を後押し（ただし中核は使命感）。
- **市民科学（Zooniverse 等）** — 銀河分類・生物観察などの分類タスクを、進捗・貢献可視化でエンゲージ。

!!! tip "クラウドソーシングが成功しやすい理由"
    これらの成功例に共通するのは、**もともと内発的動機（貢献したい・解きたい・認められたい）を持つ人**を対象に、その動機を**増幅**している点。フィランソロピスト/アチーバー/エクスプローラー（第4章）が活躍する設計。外的報酬で釣るのではなく、**有能感・関係性・意義**を可視化することが効いている。第14章で見る「健全なゲーミフィケーション」の典型でもある。

## 11.5 ビジネス分野の成功と失敗の分かれ目

!!! quote "Gartner の警告を振り返る"
    第1章で触れた「ゲーミフィケーション化アプリの80%が失敗する」という予測は、まさにビジネス分野の安直な導入ブームを背景にしていた。失敗の主因は「設計の稚拙さ」——本書の言葉で言えば、**目標から逆算せず（第5章）、PBLを貼っただけ（第8章）、単一動機を前提にし（第4章）、運用しなかった（第7章）**ことに尽きる。

| 成功しやすい | 失敗しやすい |
|---|---|
| 内発的動機を増幅（クラウドソーシング型） | 外的報酬で釣るだけ |
| 任意参加・協力ベース | 強制・個人の晒し |
| 質と量のバランスを測る | 単一の数字を競わせる |
| 長期目標・運用を伴う | 作りきり |
| ユーザー/従業員の利益と一致 | 管理・搾取の道具 |

!!! check "ビジネス・ゲーミフィケーション設計のチェック"
    - [ ] 顧客/従業員の**利益**と、ビジネス目標が**一致**しているか？
    - [ ] （労働文脈）**任意参加**で、強制・晒しになっていないか？
    - [ ] 業績評価・給与と**直結**させて外発化していないか？
    - [ ] 競わせる指標に**質の観点**が含まれ、ズルの抜け穴がないか？
    - [ ] ロイヤルティが**割引依存**になっていないか？（真のロイヤルティか）
    - [ ] 運用・更新の体制があるか？（作りきりでないか）

## この章のまとめ

- **マーケティング/ロイヤルティ**は最も確立した応用。ただし割引依存・複雑化・コストに注意。
- **人事・従業員**領域は最も慎重を要する。労働では参加が強制化しやすく、**自律性の剥奪・搾取**のリスクが固有。任意・協力・非評価連動が原則。
- **営業/コールセンター**は測定の歪みが最も出やすい。質の併用・個人ベスト・長期接続で対処。
- **クラウドソーシング/市民科学**は、内発的動機を増幅する成功例が多い（Foldit・Stack Overflow）。
- 成功と失敗を分けるのは、**ユーザー/従業員の利益とビジネス目標が一致しているか**。

次章では、環境・金融・公共など、**社会全体の行動変容**を扱う第Ⅲ部最終章へ進む。

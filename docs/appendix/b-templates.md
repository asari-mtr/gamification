## B.2 SDT 3欲求 点検シート（第2章）

各施策について、3つの基本的心理欲求への影響を ＋/0/− で評価する。

| 施策 | 有能感 | 自律性 | 関係性 | 総合判断 |
|---|---|---|---|---|
| 例:グローバルランキング | ＋（上位）/−（下位） | 0 | −（疎外） | 要セグメント化 |
| 施策1 |  |  |  |  |
| 施策2 |  |  |  |  |

!!! tip "使い方"
    どれか1つでも明確に「−」が付く施策は、その欲求を損なう副作用を持つ。代替・補完（例:競争に協力を併設）を検討する。

## B.5 メカニクス選定チェック（第6章）

```
□ 各メカニクスは 6D の目標・行動に紐づくか
□ 各メカニクスを心理欲求（第2章）で説明できるか
□ 達成系（左脳）に偏っていないか／協力・物語で補完したか
□ 競争系を使うなら下位対策（セグメント・個人ベスト・任意参加）はあるか
□ ストリーク等の損失回避系に救済はあるか
□ 経済系・変動報酬の倫理・法規制を確認したか
□ 数を絞り、少数を深く機能させているか
```

## B.7 倫理レビュー・チェックリスト（第14章 要約）

```
【透明性】
□ 仕掛けを説明してもユーザーは受け入れるか（透明性テスト）
□ 隠すことで成立する仕掛けはないか

【利益の一致】
□ ユーザーの利益と提供者の利益は一致しているか
□ 設計者自身が使いたいか／生活を改善するか（Manipulation Matrix）

【自律性】
□ やめる自由・オプトアウトがあるか
□ 離脱を不当に妨げていないか

【脆弱性・射幸性】
□ 未成年・脆弱な集団への影響を検討したか
□ 有料変動報酬（ガチャ）をユーザー利益目的の領域に持ち込んでいないか
□ 関連法規制（ギャンブル・景表法・確率表示）を確認したか

【労働文脈】
□ 参加は真に任意か／監視・強制・晒しになっていないか

【持続可能性】
□ 依存でなく内発的動機・健全な習慣へ橋渡ししているか
```

## B.8 プロジェクト着手前セルフチェック（第1・18章 統合）

```
□ この課題はゲーミフィケーションに向いているか（繰り返し・進捗可視化・自発性）
□ 問題の原因は動機か（UX・能力・環境の問題ではないか）
□ 目標→行動→人→ループ→楽しさ→要素 の順で設計したか（要素から始めていないか）
□ 成果指標と測定計画があるか
□ 倫理レビューを通したか
□ 小さく試して検証する計画になっているか
```

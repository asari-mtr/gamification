# 付録C　ケーススタディ索引

本書で取り上げた事例を、分野別に索引化する。各事例から学べる教訓と参照章を示す。成功例だけでなく、失敗・批判事例も「何を避けるべきか」の教材として収録する。

## 教育・学習

| 事例 | 分類 | 学べる教訓 | 章 |
|---|---|---|---|
| **Duolingo** | 成功（要注意点あり） | 短期/長期ループの統合、セグメント競争（リーグ）でアンチパターン回避。ただし継続≠習得。 | 第9章 |
| **Kahoot!** | 成功（用途限定） | 集団的興奮を生むが、速さ偏重・下位萎縮・内容より興奮が残る懸念。 | 第9章 |
| **Classcraft** | 応用例 | クラスのRPG化。協力・成長重視が安全、評価連動はリスク。 | 第9章 |
| **MOOC（完走率問題）** | 課題 | ゲーミフィケーションよりコホート（社会的つながり）が完走に効く。 | 第9章 |

!!! tip "ケーススタディの読み方"
    成功例は「なぜ効いたか」を本書の原則（SDT・多動機・ループ・倫理）に照らして分解する。失敗例は「どのアンチパターン（第8章）・倫理問題（第14章）に該当するか」を特定する。事例を**原則に翻訳して理解**することで、自分の文脈に応用できる教訓になる。
